Checkboxes allow the selection of multiple options from a set.

If you have multiple options appearing in a list, you can preserve space by using checkboxes instead of on/off switches.

If you have a single option, avoid using a checkbox and use an on/off switch instead.

Basic example 
<mat-checkbox [(ngModel)]="isChecked" 
        name="cb" 
        value="basic_checkbox" 
        [color]="checkboxColor()" 
        (change)="isIndeterminate = false"
        [indeterminate]="isIndeterminate" 
        [disabled]="isDisabled" 
        [align]="alignment">
  Do you want to
  <em>foobar</em> the
  <em>bazquux</em>?
</mat-checkbox>

Name Type Default Description
align @Input start Whether or not the checkbox should appear before or after the label.
aria-label @Input Attached to the aria-label attribute of the host element. In most cases, arial-labelledby will take precedence so this may be omitted.
aria-labelledby @Input Users can specify the aria-labelledby attribute which will be forwarded to the input element
id @Input A unique id for the checkbox input. If none is supplied, it will be auto-generated.
name @Input Name value will be applied to the input element if present.
required @Input Whether the checkbox is required.
value @Input The value attribute of the native input element.
checked @Attribute Whether the checkbox is checked.
indeterminate @Attribute Whether the checkbox is indeterminate. This is also known as "mixed" mode and can be used to represent a checkbox with three states, e.g. a checkbox that represents a nested list of checkable items. Note that whenever checkbox is manually clicked, indeterminate is immediately set to false.
change @Output Event emitted when the checkbox's checked value changes.
